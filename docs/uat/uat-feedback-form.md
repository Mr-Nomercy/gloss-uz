## Severity Reference (from bug-triage.md)
| Severity | Definition | SLA |
|----------|------------|-----|
| **P0 - Critical** | System down, data loss, payment failure, security breach | Fix immediately, block launch |
| **P1 - High** | Major feature broken, payment issues, data corruption | Fix within 4 hours, block launch |
| **P2 - Medium** | Feature partially working, UX issue, non-critical bug | Fix within 24-48h, can launch with workaround |
| **P3 - Low** | Cosmetic, minor UX, enhancement | Fix in next sprint/backlog |
